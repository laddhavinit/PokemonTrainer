🎮 Pokemon Trainer Application

Execution Flow Map - ng serve Command

📱 Phase 1: Application Bootstrap
1
Angular CLI starts development server
Webpack dev server starts, compiles TypeScript, and serves the application on localhost:4200
2
Load main.ts
Entry point: src/main.ts
Calls bootstrapApplication(App, appConfig)
3
Load App Configuration
Loads: src/app/app.config.ts
Sets up providers, routing, and other app-wide configurations
4
Initialize App Component
Loads: src/app/app.ts
Creates root component, sets up router outlet and header visibility logic
⬇️
🔧 Phase 2: Service Initialization (Dependency Injection)
5
PokemonService Creation Trigger
When: First component that injects PokemonService is created
Where: Angular's DI system detects private pokemonService: PokemonService in:
• catalogue.component.ts constructor
• trainer.component.ts constructor
• header.component.ts constructor
6
PokemonService Constructor Executes
File: src/app/services/pokemon.service.ts
Singleton Creation: Due to providedIn: 'root', only ONE instance is created
• Calls loadCaughtPokemonIds() - reads trainer's caught Pokemon from localStorage
• Calls loadTotalCountFromCache() - reads total Pokemon count from sessionStorage
• Initializes all signals with default values
7
Service Instance Injection
Angular DI injects the SAME PokemonService instance into all components that request it
Shared State: All components share the same service data (signals, cache, etc.)
⬇️
🛣️ Phase 3: Initial Route Resolution
7
Router Navigation Starts
Angular Router attempts to navigate to the initial route (usually '/')
8
AuthGuard Execution
Executes: src/app/guards/auth.guard.ts
• Checks if running in browser (SSR safety)
• Reads trainerName from localStorage
• Determines authentication status
⬇️
🔀 Phase 4: Route Decision Branches
✅ User IS Logged In
9a
Check Target Route
If trying to access landing page ('') → Redirect to '/catalogue'
If accessing '/trainer' or '/catalogue' → Allow access
10a
Load Target Component
For Catalogue: catalogue.component.ts
For Trainer: trainer.component.ts
⚡ PokemonService gets created HERE when component constructor runs
❌ User NOT Logged In
9b
Check Target Route
If trying to access landing page ('') → Allow access
If accessing '/trainer' or '/catalogue' → Redirect to '/'
10b
Load Landing Component
Loads: landing.component.ts
Shows trainer name input form
⬇️
🔄 Phase 5: Component Lifecycle & Data Loading
11
Component Constructor Execution
🎯 THIS IS WHERE PokemonService GETS CREATED!
When component constructor runs: constructor(private pokemonService: PokemonService)
Angular DI system creates PokemonService instance and injects it into the component
12
ngOnInit Lifecycle Hook
Landing Component: Sets up form and validation
Catalogue Component: Calls pokemonService.refreshCaughtPokemonIds() and loadCurrentPage()
Trainer Component: Calls pokemonService.refreshCaughtPokemonIds() (effect handles loading)
13
API Calls (if needed)
Catalogue: loadPokemonPage() → Fetches Pokemon list from PokeAPI or cache
Trainer: getCaughtPokemon() → Fetches caught Pokemon details from PokeAPI or cache
⬇️
🎨 Phase 6: Template Rendering & UI Display
14
Header Component Rendering
header.component.ts renders if not on landing page
Shows navigation, trainer name, and Pokemon count
15
Main Component Template Rendering
Component template renders with data from services
Angular change detection updates the DOM
16
Application Ready
🎉 Application is fully loaded and interactive!
User can now interact with the Pokemon Trainer application
